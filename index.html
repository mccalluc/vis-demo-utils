For Refinery visualizations to run from GH Pages, some boilerplate is required. This project provides a single file, utils.js, which when included sets up error handling, parses the url, and dispatches to the specified vis.

In production, all the same things need to be done, but they will probably be done differently: requirejs and other dependencies will be local copies instead of pulled from a CDN, and errors might be logged to the server. The code here might serve as a checklist, but should not be copied verbatim into Refinery.
